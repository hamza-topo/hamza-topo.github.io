Implementing Observers in Laravel

By Hamza Ait Sidi Said

Laravel, a powerful PHP framework, provides developers with various tools and features to streamline the development process. One such feature is the Observer pattern, which allows you to decouple different parts of your application, making it more modular and maintainable.

Understanding Observers

In Laravel, observers are classes that listen for specific events that occur in your Eloquent models. These events can include creating, updating, deleting, and other model-related actions. By using observers, you can perform additional tasks or actions in response to these events without cluttering your models with unnecessary logic.

Creating an Observer

To create an observer in Laravel, start by generating a new observer class. You can use the following artisan command:

php artisan make:observer MyObserver

This command will create a new observer class named MyObserver in the App/Observers directory.

Open the generated observer class and define the methods for the events you want to observe. For example:

php
  class MyObserver
  {
      public function creating(Model $model)
      {
          // Logic to perform when a model is being created
      }
  
      public function updating(Model $model)
      {
          // Logic to perform when a model is being updated
      }
  
      // Other event methods can be added as needed
  }

Registering an Observer

Once you've created your observer, you need to register it with the model you want to observe. You can do this by adding the observer to the model's observe array:

php
  class YourModel extends Model
  {
      protected $dispatchesEvents = [
          'creating' => YourModelCreating::class,
          'updating' => YourModelUpdating::class,
          // Other model events can be added as needed
      ];
  }

With this setup, the specified observer will now listen for the defined events on the YourModel model.

Conclusion

Observers in Laravel provide a clean and organized way to handle model events. By separating the logic related to events into observer classes, you can keep your models focused on representing data and improve the overall maintainability of your Laravel applications.

Deploying Your Laravel Application on Heroku

By Hamza Ait Sidi Said

Heroku makes it easy to deploy and manage Laravel applications in a cloud environment. In this guide, we'll walk through the steps to deploy your Laravel app on Heroku using the Heroku Command Line Interface (CLI).

Prerequisites

Before you begin, ensure you have the following:

  • A Heroku account (sign up at Heroku Sign Up).
  • Heroku CLI installed (download it from Heroku CLI).
  • Your Laravel application code hosted on a version control system (e.g., Git).

Deployment Steps

Follow these steps to deploy your Laravel application on Heroku:

  1. Create a Heroku App: In your terminal, navigate to your Laravel project directory and run the following command to create a new Heroku app:
    2. heroku create your-app-name
  2. Configure Environment Variables: If your Laravel app uses environment variables (e.g., for database credentials), set them on Heroku:
    3. heroku config:set APP_KEY=$(php artisan --no-ansi key:generate --show)

    Replace your-app-name with your desired app name.

  3. Set the Application Environment: Set the environment to production on Heroku:
    4. heroku config:set APP_ENV=production
  4. Add a Procfile: Create a file named Procfile (without any file extension) in your Laravel project root with the following content:
    5. web: vendor/bin/heroku-php-apache2 public/
  5. Commit Your Changes: Make sure your changes are committed to your version control system (e.g., Git).
  6. Deploy Your Application: Deploy your Laravel application to Heroku:
    7. git push heroku master
  7. Run Migrations: Run database migrations on Heroku:
    8. heroku run php artisan migrate
  8. Open Your App: Once the deployment is complete, open your app in the browser:
    9. heroku open

Conclusion

Congratulations! Your Laravel application is now live on Heroku. Heroku's simplicity and scalability make it an excellent choice for hosting and managing your Laravel projects in a cloud environment.

Real-Time Communication with Pusher WebSocket and Laravel

By Hamza Ait Sidi Said

Real-time communication has become a crucial aspect of modern web applications. Laravel, a powerful PHP framework, makes it easy to integrate real-time features using Pusher, a hosted WebSocket service. In this tutorial, we'll explore the steps to set up and use Pusher WebSocket with a Laravel application.

Prerequisites

Before you begin, ensure you have the following:

  • A Laravel project set up and running.
  • A Pusher account (sign up at Pusher).

Setting Up Pusher in Laravel

Follow these steps to integrate Pusher with your Laravel application:

  1. Install the Pusher PHP Library: Run the following command in your terminal:
    2. composer require pusher/pusher-php-server
  2. Configure Pusher Credentials: Add your Pusher credentials to the .env file:
    3. PUSHER_APP_ID=your-app-id
  PUSHER_APP_KEY=your-app-key
  PUSHER_APP_SECRET=your-app-secret
  PUSHER_APP_CLUSTER=your-app-cluster
  3. Install Laravel Echo and Pusher JavaScript SDK: Run the following commands:
    4. npm install --save laravel-echo pusher-js
  4. Configure Broadcasting: Set the broadcasting driver to Pusher in the config/broadcasting.php file: 
    5. 'default' => env('BROADCAST_DRIVER', 'pusher'),
  5. Create an Event: Generate an event using the artisan command:
    6. php artisan make:event YourEvent
  6. Update Event Logic: Open the generated event file in app/Events/YourEvent.php and define the logic to broadcast: 
    7. public function broadcastOn()
  {
      return new Channel('your-channel');
  }
  7. Broadcast the Event: Trigger the event where needed in your application:
    8. event(new YourEvent($data));

Listening to Events with Laravel Echo

Use Laravel Echo to listen to events on the client side:

  1. Install Laravel Echo and Pusher JavaScript SDK: If not done already, run the following commands:
    2. npm install --save laravel-echo pusher-js
  2. Configure Laravel Echo: In your JavaScript file (e.g., resources/js/bootstrap.js), configure Laravel Echo: 
    3. import Echo from 'laravel-echo';
  
  window.Pusher = require('pusher-js');
  
  window.Echo = new Echo({
      broadcaster: 'pusher',
      key: process.env.MIX_PUSHER_APP_KEY,
      cluster: process.env.MIX_PUSHER_APP_CLUSTER,
      forceTLS: true,
  });
  3. Listen for Events: In your Vue component or JavaScript file, listen for the events: 
    4. mounted() {
      Echo.channel('your-channel')
          .listen('YourEvent', (data) => {
              console.log('Event received:', data);
              // Handle the event data as needed
          });
  }

Conclusion

Congratulations! You've successfully set up Pusher WebSocket with your Laravel application. Real-time communication opens up possibilities for interactive and dynamic user experiences in your web applications.
